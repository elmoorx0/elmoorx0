Elmoorx API Reference

Complete documentation for the Elmoorx Framework v1.0 runtime, compiler, and CLI.

Bundle Size
The entire runtime is ~4kb gzipped — smaller than a single React component. Only APIs you actually use are bundled; the rest are tree-shaken away.

Installation

Install the Elmoorx CLI globally:

terminal
npm install -g @elmoorx/cli # or pnpm add -g @elmoorx/cli yarn global add @elmoorx/cli

Quick Start

Create a new project and start the dev server:

terminal
elmoorx create my-app cd my-app npm install elmoorx dev # → http://localhost:3000

$state

Creates a reactive signal. Reads track; writes trigger effects. The fundamental reactive primitive in Elmoorx.

function $state<T>(initial: T): Signal<T>

Example

example.tsx
import { $state } from "@elmoorx/runtime"; const count = $state(0); count() // → 0 count.set(5) // triggers effects count.set(c => c + 1) // updater function

$computed

Creates a derived value that auto-updates when its dependencies change.

function $computed<T>(fn: () => T): Computed<T>

Example

const price = $state(100); const qty = $state(2); const total = $computed(() => price() * qty()); total() // → 200 price.set(150) total() // → 300 (auto-updated)

$effect

Runs a side effect whenever its dependencies change. Returns a cleanup function.

function $effect(fn: () => void | (() => void)): () => void

Example

$effect(() => { document.title = `Count: ${count()}`; }); // Cleanup const dispose = $effect(() => { const id = setInterval(() => console.log(count()), 1000); return () => clearInterval(id); }); dispose(); // stop interval

$store

Creates a deep reactive proxy. Mutations like store.todos.push(item) or store.user.name = "X" auto-trigger surgical DOM updates — no setters, no immutability boilerplate.

function $store<T extends object>(initial: T): Store<T>

Example

const store = $store({ user: { name: "Alice", age: 30 }, cart: [], theme: "dark" }); // All mutations are reactive store.cart.push({ id: 1, qty: 2 }); store.user.name = "Bob"; store.theme = "light"; // Deep mutations work too store.user.age = 31; // triggers update

island()

Marks a component as a client-side island. The server renders the component's initial HTML; the client only boots JavaScript for islands explicitly wrapped with island(). Everything else is pure HTML.

function island<P>(component: (props: P) => ElmoorxNode): IslandComponent<P>

Example

import { $state, island, h } from "@elmoorx/runtime"; const Counter = island(() => { const count = $state(0); return <button onClick={()=>count.set(c=>c+1)}> Count: {count} </button>; }); // SSR: renders
Count: 0
 // Client: boots ONLY this island — zero hydration elsewhere

$html

Explicitly-trusted HTML. Auto-sanitizes scripts, iframes, on* handlers, and javascript: URLs. There is no implicit path — XSS protection is on by default.

function $html(input: string): { __html: string; __trusted: true }

Example

// ❌ Auto-escaped — no XSS surface <div>{userInput}</div> // ✅ Explicit trusted HTML — auto-sanitized <div>{$html(trustedContent)}</div>

h() — JSX Factory

The JSX factory. The compiler converts JSX to h() calls.

function h( tag: string | Function, props: Record<string, unknown> | null, ...children: ElmoorxNode[] ): ElmoorxNode

Example

// JSX <div className="card" onClick={handleClick}> Hello {name} </div> // Compiles to h("div", { className: "card", onClick: handleClick }, "Hello ", name)

SSR & Streaming

Elmoorx streams server-rendered HTML on first request. Islands are hydrated client-side; everything else is pure HTML.

import { renderToString } from "@elmoorx/runtime"; const html = renderToString(App()); // → "
Hello
"

Routing

File-based routing. Convention:

  • src/index.elmoorx.tsx → /
  • src/about.elmoorx.tsx → /about
  • src/users/[id].elmoorx.tsx → /users/:id
  • src/[...catch].elmoorx.tsx → /* (catch-all)

Security

Security headers are auto-applied to every response. No configuration required.

Auto-applied headers
  • Content-Security-Policy: default-src 'self'
  • X-Frame-Options: DENY
  • X-Content-Type-Options: nosniff
  • Strict-Transport-Security: max-age=31536000
  • Referrer-Policy: strict-origin-when-cross-origin
  • Permissions-Policy: geolocation=(), microphone=(), camera=()
  • CSRF Token: per-request, auto-validated

Deployment

Elmoorx deploys to any edge platform. The runtime is 4kb — well under every platform's bundle limit.

terminal
elmoorx deploy # Or platform-specific: elmoorx deploy --target=cloudflare elmoorx deploy --target=vercel elmoorx deploy --target=deno elmoorx deploy --target=node

elmoorx generate

AI Copilot — generates production-ready components from natural language descriptions.

terminal
elmoorx generate "login form" elmoorx generate "todo list with filters" elmoorx generate "data table with sorting" elmoorx generate "navbar with mobile menu" elmoorx generate "modal dialog" elmoorx generate "contact form with validation" elmoorx generate "search bar with debounce" # Set OPENAI_API_KEY for AI-powered generation export OPENAI_API_KEY=sk-... elmoorx generate "user profile with avatar upload"
Template Engine
Without an API key, the Copilot uses a built-in template engine that recognizes common patterns (login, todo, counter, form, chart, modal, navbar, table, search, dropdown). Set OPENAI_API_KEY to enable full AI-powered generation.
