Why Elmoorx?

⚡

Zero-Hydration Islands

Only interactive components ship JavaScript. Everything else is static HTML — zero hydration cost.

🔒

Auto-Security

Sanitization, CSRF, CSP, HSTS built-in. No dangerouslySetInnerHTML to forget.

📡

Reactive Store

Proxy-based deep reactive container. Mutations are automatically tracked — no setters, no immutability boilerplate.

🌍

Edge-Ready

~4.2kb gzipped, runs anywhere — Cloudflare, Vercel, Deno, Node. 28ms cold start.

🎯

Signals-Based

Surgical DOM updates at property granularity. No vdom diffing, no re-render storms.

📦

78 Packages

Runtime, router, server, compiler, CLI, UI (648 components), auth, postgres, i18n, web3, AR/VR, and more.

Quick Start

npm install @elmoorx/runtime @elmoorx/router @elmoorx/server

# Or use the CLI
npx @elmoorx/cli create my-app
cd my-app
npm install
npm run dev
# → http://localhost:3000
import { $state, $computed, $effect, h } from "@elmoorx/runtime";

const count = $state(0);
const doubled = $computed(() => count() * 2);

$effect(() => {
  document.title = `Count: ${count()}`;
});

count.set(1);       // triggers effect
count.set(c => c + 1); // updater function
import { island, h, $state } from "@elmoorx/runtime";

const LikeButton = island(() => {
  const likes = $state(0);
  return h("button",
    { onClick: () => likes.set(l => l + 1) },
    "♥ ", () => likes()
  );
});

// Server renders static HTML; client boots only this island
// Zero JS shipped for the rest of the page

API Reference

📡 Signals

  • $state<T>(initial)
  • $computed<T>(fn)
  • $effect(fn)
  • $batch(fn)

🗄️ Store

  • $store<T>(initial)
  • store.$subscribe(fn)
  • store.$serialize()
